# Token Surface Checklist

- Token source remains `tokens/zdp.tokens.json`.
- Generated token names and CSS variables match the source.
- Color entries include hex fallback and OKLCH intent.
- Token names are role-based rather than product-specific.
- Optional font CSS entries stay separate.
- Flat UI rules remain intact: no decorative shadows, gradients, shimmer, hover movement, or oversized pill defaults.
