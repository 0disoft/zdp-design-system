# Component Boundary Checklist

- Component props describe UI state and accessibility, not product authority.
- Consumers own routing, data fetching, mutation, permissions, locale fallback, ads policy, and destructive action execution.
- User-visible default text is natural and overrideable.
- Focus, keyboard, reduced motion, semantic markup, and screen reader names are preserved.
- New component behavior has story, fixture, type, package, and docs impact review.
